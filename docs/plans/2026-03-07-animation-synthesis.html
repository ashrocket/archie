Chapter One

The Three Scenarios

All four teams independently designed for the same three animation behaviors. No coordination. Same conclusions.

The Journey Response

Archie gets up from the chat bubble, walks off-screen, wanders down a wooded trail with parallax scrolling, sits on a log to think. When the response is ready, he writes it on a paper airplane, throws it — the airplane arcs back to the chat window, lands, unfurls, and the text is readable. Wait-time becomes anticipation.

The Impatient Crawler

While the user types, Archie crawls around the edges of the chat window frame. He hangs upside-down from the top, peeks around corners, taps his foot on the border, adjusts his glasses anxiously. His speed correlates to typing speed — faster typing means a more excited Archie.

The Idle Dreamer

During quiet time, Archie gradually falls asleep — head nods, glasses slip, Z's float up. Alternatively, he notices the document you're chatting about, gestures at it, conjures a cartoon explainer, then lassoes it into the chat where it becomes a downloadable artifact.

Chapter Two

Where All Four Teams Converge

When independent teams reach the same conclusion without coordinating, those conclusions are real.

Shared Character Insights

  • Glasses adjustment is Archie's signature tell — thinking, waking up, nervous
  • Expression changes snap with a brief hold (the "Newgrounds snap"), never tween smoothly
  • The walk cycle should feel stiff and deliberate — storybook puppet, not Disney fluid
  • Sleep progression: fidget → drowsy → head nod → asleep → Z's
  • The glasses-push-up on wake is universally the most endearing animation beat

Shared Architecture

  • The chat widget remains a single droppable <script> tag
  • Bridge server already serves static files — no server changes for assets
  • Archie breaks out of the chat panel bounds via position: fixed layers outside the panel DOM
  • The paper airplane is the signature moment — every team spent the most design energy here
  • prefers-reduced-motion skips to final state
  • Users need a skip mechanism (click to jump to text)

Canvas

Canvas 2D + Sprite Sheets

Raw Canvas API with modular body-part sprite sheets composited at render time. Zero dependencies.

  • Zero external dependencies
  • Precise pixel control
  • 24fps cap as deliberate aesthetic
  • Combinatorial sprite expressiveness
  • Opaque to screen readers
  • New animations = new art assets
  • Must build easing/timelines from scratch
Engine

Canvas 2D + State Machine

Three-canvas layer architecture with 5-level priority interrupt system and three-tier asset loading.

  • Rigorous state management
  • Dirty-rect rendering optimization
  • Adaptive frame rate for mobile
  • Three-tier lazy asset loading
  • Same Canvas accessibility gap
  • Complex custom RAF management
  • Heavy total asset weight (~465 KB)
Soul

CSS + DOM (Evolve Existing)

Build on the CSS character already in ArchieReadMe.html. Zero asset loading, fully accessible by default.

  • CSS Archie already exists (200 lines)
  • Zero asset downloads
  • Naturally accessible (DOM = screen readers)
  • Trivially responsive
  • Can't do complex walk cycles
  • Limited to transform/opacity
  • No visual richness of sprites
Kinetic

SVG + GSAP + Lottie

Motion graphics approach: GSAP labeled timelines, Lottie for complex sequences, SVG character rig. Designer-controlled pipeline.

  • Lightest weight (111 KB gzipped)
  • SVG is DOM → ARIA accessible
  • GSAP labels: seek, scrub, reverse
  • After Effects → Lottie creative pipeline
  • MorphSVG for paper airplane fold
  • Resolution independent at any DPI
  • External deps: GSAP (~38KB), Lottie (~22KB)
  • MorphSVG/DrawSVG are paid plugins
  • Requires After Effects for authoring
^ the rendering debate was the juiciest part
Chapter Four

Standout Ideas by Team

Each team brought something unique that the others didn't think of.

Canvas

Modular Skeletal Sprites

Instead of one giant sprite sheet, Archie is assembled from body-part sheets at render time: head, torso, arms, legs, glasses. A small set of parts creates hundreds of poses through combination — walking-while-looking-left, sitting-while-glasses-slip, crawling-while-worried. The slight imperfections in part alignment make him feel more hand-puppeted.

Canvas

Verlet Rope Physics for the Lasso

The lasso rope is modeled as a chain of 20 point-masses with distance constraints. Gravity and damping applied to all points. The twirl is a force on the leading mass. The rope is rendered by stroking a bezier path through the points. Physically plausible, visually hand-drawn.

Engine

Priority Interrupt State Machine

A 5-level priority system prevents jarring animation conflicts:

// Priority 5: AIRPLANE_FOLD, FLIGHT, UNFURL — NEVER interruptible // Priority 4: EXPLAINER, LASSO — fast-forward on interrupt // Priority 3: WALKING, SITTING, THINKING — graceful exit animation // Priority 2: TYPING_DETECTED, CRAWLING — yield to user action // Priority 1: IDLE, SLEEPING, DREAMING — freely interruptible
Engine

Three-Tier Asset Loading

Critical (loaded with widget): idle sprite, ~45KB. Eager (after first paint): walking + thinking + forest, ~170KB. Lazy (on first need): airplane, sleeping, crawling, lasso, ~200KB. The chat is functional before any animation assets arrive.

Kinetic

Custom Easing Language

Three signature curves define Archie's motion personality across every animation:

archieEase — soft overshoot (12%) with one gentle wobble — "hand-drawn character moving through physical space" archieSnap — fast attack, holds overshoot, snaps to final — the Newgrounds puppet quality archieSettle — slow deceleration with micro-bounce — for landing, sitting down, coming to rest
Kinetic

GSAP Master Timelines with Labels

The entire journey response is one scrubable, seekable timeline:

journey.addLabel("anticipation", 0) .addLabel("glasses-bounce", 0.35) .addLabel("walk-off", 0.85) .addLabel("scene-transition", 1.6) .addLabel("forest-arrive", 2.2) .addLabel("writing", 3.0) .addLabel("airplane", 3.8) .addLabel("unfurl", 5.5) // User clicks skip: journey.seek("unfurl") // Response arrives fast: journey.timeScale(2)

↑ This declarative choreography is impossible with raw Canvas requestAnimationFrame loops.

The Emotional State Map

Team SOUL mapped every user interaction to an Archie emotional response. He is a character with interiority, not a progress indicator.

While You Type

Typing fast
Excited anticipation. Leans forward, bouncing on toes. "This person has something to say."
Typing slowly
Patient attention. Head tilts, hand on chin. Blink rate slows.
Deleting a lot
Mild concern. Head tilts, eyebrows draw together. Adjusts glasses nervously.
Long pause
Curiosity. Peers toward the text area. One eyebrow raised. "What are they thinking about?"
Clears input
Brief confusion. Double-blink. Slight shrug. Returns to neutral.
Pastes large text
Surprise, then readiness. Eyes widen. Reaches for notebook. Straightens posture.

Deep research. Pages flip. Looks up at the sky. The world slows down. The scene breathes.

Worried. Looks around nervously. Taps the dialogue frame. Tries to fix something with his hands.

Chapter Seven

The Vivacity System

Team SOUL's most important innovation. Clippy failed because it never learned to shut up. Archie must be the anti-Clippy.

Animation intensity operates on a 0–100 scale that automatically decays as you use it more. The first message gets the full storybook. By message 25, Archie matches your pace. He earns your trust by reading the room.

THE DELIGHT CURVE

Msg 1
100
Full storybook wonder. The journey. The airplane. Everything.
Msg 5
75
Warm companion. Crawler at reduced frequency. Animations familiar.
Msg 15
50
Efficient partner. Journey on long responses only. Minimal expressions.
Msg 25+
40
Trusted colleague. Abbreviated journey. He's part of the workspace now.

How It Adapts

Signals that boost vivacity (they're enjoying it): hovering over Archie during animation (+5), clicking Archie in idle (+10), interacting with a lassoed element (+10), returning after 24+ hours (reset to 60).

Signals that reduce vivacity (they want less): 3+ messages in 30 seconds (-20), clicking skip during journey (-10), dismissing an animation with Escape (-10), navigating away during journey (-5).

The escape hatch: A gear icon in the chat header opens a vivacity slider. One end says "Text Only" and the other says "Full Storybook." Manual override persists in localStorage.

Chapter Nine

The Recommended Synthesis

Take the best from each team. Build one system.

KINETIC's Rendering + SOUL's Brain + ENGINE's Spine

SVG + GSAP + Lottie for rendering. The Vivacity System for emotional intelligence. Priority interrupt state machine for animation safety.

Why KINETIC Wins the Rendering Debate

  1. Lightest weight — 111KB gzipped vs 380KB+ for Canvas sprites
  2. Best accessibility — SVG is DOM, therefore ARIA-reachable. Canvas is an opaque rectangle to screen readers.
  3. Best creative pipeline — After Effects → Bodymovin → Lottie. A motion designer controls quality, not a programmer.
  4. Best choreography — GSAP labeled timelines with seek, scrub, reverse, timeScale. Impossible with raw Canvas RAF loops.
  5. Resolution independent — SVG scales to any DPI. No 2x/3x sprite variants.
  6. Best paper airplane — SVG MorphSVG path morphing is objectively more elegant than sprite-sheet frame flipping.
"Archie is a performer, not a player character. He walks on stage, does his bit, delivers the line, and waits in the wings. This is theater. This is editorial. The motion graphics stack — GSAP, Lottie, SVG, CSS — is the same stack used by the NYT Interactives team, by Stripe's homepage, by Apple's product pages. These are the highest-quality web animations in existence."
— Team KINETIC

Why SOUL's Vivacity System Is Non-Negotiable

Without it, Archie becomes Clippy within 10 minutes. The auto-decay from wonder to efficiency across a session, the boost/decay signals from user behavior, and the manual override slider are the difference between a character that delights and one that annoys.

Why ENGINE's Priority Interrupts Hold It Together

Without state management, animations collide. What happens when the user sends a message while Archie is falling asleep? What if the response arrives during the walk? The 5-level priority system ensures response delivery is never interrupted, background states yield instantly, and everything else gracefully transitions.
